# 민턴LIVE 관리자 모바일 UX 개선 검토 요청서

아래 내용을 그대로 미토스에게 복사해서 전달하면 됩니다.  
파일은 `index.html`, `checkin.html` 두 개를 함께 업로드해 주세요.

---

콕매치 `민턴LIVE`의 **관리자 페이지 모바일 UX**만 집중 검토해줘.

## 0. 현재 버전과 검토 범위

- 앱 버전: `v1.10.149`
- 배포/확인 URL: `https://kimminhyun22.github.io/badminton/?v=1.10.149&adminux=review&fresh=1`
- 핵심 파일:
  - `index.html`: 관리자 페이지, 민턴LIVE 운영 로직/화면
  - `checkin.html`: 회원용 출석/상태 링크
- 검토 제외:
  - 월례 팀전 전체 라운드 대진표
  - 기존 팀전 청/홍 정합성
  - 앱스토어/플레이스토어 패키징
  - 디자인 취향만 바꾸는 장식성 리디자인

이번 요청의 핵심은 **관리자 페이지가 너무 복잡해서 모바일에서 무엇을 해야 하는지 한눈에 들어오지 않는다**는 점이야.

민턴LIVE의 목표 흐름은 다음과 같아.

`공지 → 투표 → 현장 출석/대기 → 자동 시작 → 상태 변경/예약 → 자동 대진`

완전자동화 관점에서 관리자는 운동 중 휴대폰으로 복잡한 편집을 하는 사람이 아니라, **대시보드만 보고 특이사항만 처리하는 사람**이어야 해.

## 1. 사용자/운영 환경

- 실제 사용자는 배드민턴 동호회 총무 또는 운영자.
- 운동 중 모바일로 확인한다.
- 3~4코트, 20~35명 정도를 가정한다.
- 40~50대 회원도 많아 글자, 버튼, 상태 구분이 작거나 복잡하면 바로 불편하다.
- 관리자는 운동 중이거나 코트 옆에서 바쁘게 움직이므로 한 손 조작, 빠른 판단, 실수 방지가 중요하다.
- 일반 회원은 `checkin.html`에서 참석, 시작/종료, 레슨/휴식/마지막, 게임 신청 정도만 해야 한다.

## 2. 현재 체감 문제

관리자 페이지가 기능은 많지만, 지금은 다음 문제가 있어.

1. 첫 화면에서 무엇이 제일 중요한지 눈에 잘 안 들어온다.
2. 대시보드, 임박대기, 예비대기, 진행중 경기, 출석 링크, 요청, 결과, 선수 상태, 비상 조작이 섞여 있어 정신없다.
3. “지금 내가 눌러야 할 버튼”과 “그냥 참고용 정보”가 같은 무게로 보인다.
4. 모바일에서 스크롤이 길고, 상세 조작 영역이 운영 흐름을 방해한다.
5. 완전자동화를 목표로 한다면 관리자는 대시보드 중심으로 봐야 하는데, 현재는 상세 관리 앱처럼 보인다.

## 3. 반드시 지켜야 할 제품 방향

민턴LIVE 관리자 화면은 아래 우선순위로 재구성되어야 한다.

1. **오늘 운영 상태 요약**
   - 자동 시작 켜짐/꺼짐
   - 사용 코트 수
   - 라이브 인원/자유 대기/레슨·휴식/확인 필요 요청
   - 특이사항 없음 또는 처리 필요

2. **지금 바로 할 일**
   - 빈 코트가 있으면 어느 코트에 누구를 넣을지
   - 임박대기 부족 여부
   - 회원 요청 확인 필요 여부
   - 예약 대진 반영 가능 여부

3. **코트 진행**
   - 진행 중 코트 현황
   - 경기 종료 버튼
   - 종료 임박 표시

4. **임박대기/예비대기**
   - 다음 들어갈 팀이 가장 잘 보여야 함
   - 이름과 코트 투입 흐름이 명확해야 함

5. **상세 조작/비상 조작**
   - 기본 접힘
   - 필요할 때만 펼침
   - 선수 교체, 대기표 재생성, 링크 복사, 초기화 등

## 5. 꼭 답변해줘야 할 질문

아래 순서대로 답변해줘.

1. **P0/P1 버그가 있는가?**
   - 관리자 화면 단순화 전에 반드시 고쳐야 할 상태 전환/대기표/자동시작 버그가 있는지.
   - 회원 요청 처리, 대기표 정합성, 경기 종료 후 자동 보충이 깨질 가능성이 있는지.

2. **관리자 모바일 UX가 왜 복잡하게 느껴지는가?**
   - 현재 화면에서 정보 우선순위가 잘못된 부분을 구체적으로 짚어줘.
   - 어떤 카드/버튼/문구가 중복인지.
   - 어떤 영역을 기본 접힘으로 보내야 하는지.

3. **관리자 첫 화면을 어떻게 재구성해야 하는가?**
   - “한 화면에서 바로 판단 가능한” 구조를 제안해줘.
   - 상단 대시보드에 남길 정보와 제거할 정보를 구분해줘.
   - 빈 코트, 임박대기, 확인 필요 요청, 자동시작 상태를 어떻게 보여줘야 하는지.

4. **모바일에서 버튼/문구를 어떻게 줄여야 하는가?**
   - 버튼 수를 줄이는 방법.
   - 문구를 짧게 바꾸는 방법.
   - 위험 버튼과 일반 버튼을 구분하는 방법.
   - 40~50대 사용자 기준으로 글자 크기, 대비, 터치 영역이 충분한지.

5. **실제 코드 수정안을 제시해줘.**
   - 가능하면 `index.html` 기준으로 바로 적용 가능한 diff 또는 수정 방향을 줘.
   - 큰 리팩터링보다 오늘 바로 테스트할 수 있는 범위의 단계적 개선안을 우선해줘.

## 6. 내가 원하는 개선 방향 예시

이건 예시일 뿐이니 더 좋은 구조가 있으면 제안해줘.

### 상단

- `민턴LIVE 운영 중`
- 자동 시작 상태: `자동 시작 꺼짐/켜짐`
- 코트: `3코트`
- 지금 상태: `특이사항 없음` 또는 `처리 필요 2건`

### 바로 아래

- `지금 할 일`
  - 빈 코트 있으면: `1코트 비어 있음 · 다음 대기 투입`
  - 요청 있으면: `회원 요청 2건 확인`
  - 대기 부족이면: `임박대기 부족 · 대기자 확인`

### 그 다음

- `코트 현황`
  - 코트별 진행/종료 버튼

### 그 다음

- `다음 대기`
  - 임박대기
  - 예비대기 1팀

### 접힘 처리

- 출석 링크 공유/종료
- 상세 선수 목록
- 수동 선수 추가
- 대기표 강제 재생성
- 비상 선수 교체
- 결과/랭킹

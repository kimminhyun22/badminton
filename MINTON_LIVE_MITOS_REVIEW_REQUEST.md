## 0. 현재 버전과 검토 범위

- 앱 버전: `v1.10.148`
- 배포/확인 URL: `https://kimminhyun22.github.io/badminton/?v=1.10.148&mitos=review&fresh=1`
- 검토 대상: 오늘운동/민턴LIVE 자동 운영 흐름
- 검토 제외: 월례 팀전, 기존 전체 라운드 대진표, 팀전 청/홍 정합성

민턴LIVE의 목표 흐름은 다음과 같아.

`공지 → 투표 → 현장 출석/대기 → 자동 시작 → 상태 변경/예약 → 자동 대진`

완전자동화 관점에서 관리자는 대시보드만 보고 특이사항만 처리하고, 회원은 링크에서 최소 조작만 하도록 설계하는 것이 목표야.

## 3. 반드시 답변해줘야 할 질문

아래 순서대로 답변해줘.

1. P0/P1 버그가 있는가?
   - 회원이 `참석`, `시작`, `종료`, `레슨`, `휴식`, `마지막`을 눌렀을 때 관리자 상태와 대기표가 잘 반영되는가?
   - `시작/종료` 토글이 다시 깨질 가능성은 없는가?
   - 경기중 또는 임박대기 포함 선수가 상태를 바꿀 때 대기표가 깨지지 않는가?

2. 완전자동화 관점에서 흐름이 끊기는 지점은 어디인가?
   - 현재 자동 시작은 기본 off이고, 관리자가 `가능한 코트 시작` 또는 `자동 시작`을 켜야 함.
   - 이 구조가 실전 안전성과 자동화 사이에서 적절한가?
   - “관리자는 대시보드만 보고 운영”이라는 목표에 비해 여전히 손이 많이 가는 지점은 무엇인가?

3. 예약 대진 자동화가 충분한가?
   - 회원이 링크에서 파트너/상대 신청을 보냄.
   - 현재 예약 신청은 관리자 승인 후 예약 대진으로 들어감.
   - 4명 모두 `시작` 상태이고 충돌이 없으면 자동 승인/자동 반영해도 되는가?
   - 자동 승인하면 위험한 케이스는 무엇인가?

4. 관리자 UX가 모바일 운영에 적합한가?
   - 대시보드 카드가 너무 많거나 적지는 않은가?
   - “특이사항 없음”과 “처리 필요”의 대비가 충분한가?
   - 비상 조작, 상세 관리, 회원 링크, 결과, 전체 선수 상태의 접힘 구조가 적절한가?
   - 운동 중 한 손 조작 기준으로 버튼 위치/문구/크기가 적절한가?

5. 회원 링크 UX가 충분히 단순한가?
   - 회원이 본인 이름을 찾고 `참석`, `시작/종료`만 누르면 된다는 점이 명확한가?
   - 다른 사람 상태가 궁금할 수 있는데, 지금처럼 전체 명단을 보여주는 것이 좋은가?
   - “내 카드 우선 노출” 또는 “최근 선택한 내 카드 상단 고정”이 필요한가?
   - `레슨·휴식·마지막`, `게임 신청(선택)`은 접힘으로 충분한가?

6. 링크 공유가 제대로 설계되어 있는가?
   - `dailyShareCheckinLink()`가 만든 링크가 관리자/회원 흐름에 맞는가?
   - 참석투표 마감 정보가 회원에게 충분히 보이는가?
   - 링크 종료 시 사용자 경험이 적절한가?
   - 이미 실시간 현황/event가 붙은 링크가 출석 링크와 자연스럽게 이어지는가?

7. 모바일 가독성과 시인성 개선점은?
   - 40~50대 회원이 운동 중 스마트폰으로 볼 때 이름, 버튼, 코트, 다음 대기 정보가 충분히 큰가?
   - 관리자 대시보드는 현장 인원/코트/임박대기/확인/자유대기를 한눈에 볼 수 있는가?
   - 불필요한 문장이나 버튼이 아직 남아 있는가?

8. 자동화 시뮬레이션 관점에서 빠진 케이스는?
   - 30명 출석, 3~4코트, 7시 시작, 8시30분 피크, 9시 이후 감소
   - 레슨자 12~15명, 레슨 10분 + 전후 준비/정비 10분씩
   - 중간 귀가, 마지막 경기 요청, 휴식, 예약 대진 요청
   - 코트가 동시에 2~4개 비는 경우
   - 대기 가능 선수가 4명 미만으로 떨어지는 마감 구간

## 4. 내가 특히 의심하는 지점

아래는 코드 근거를 보고 중점 검토해줘.

- `자동 시작`은 완전자동화를 원하면 기본값을 켜야 할 수도 있지만, 실전 안전성을 위해 기본 off가 나을 수도 있음. 어떤 UX가 맞는지 판단 필요.
- 예약 대진은 현재 관리자 승인 구조라 안전하지만, “완전자동화” 목표와는 약간 다름.
- 회원 링크는 전체 명단이 보여서 다른 사람도 눌렀는지 확인 가능하지만, 운동 중에는 정보량이 많을 수 있음.
- 관리자 화면은 대시보드 우선으로 바뀌었지만, DOM 순서는 상세 관리가 먼저라 접근성/스크린리더/텍스트 추출 기준에서는 어색할 수 있음.
- 출석투표 마감 후 `참석자만 남기기`는 관리자가 눌러야 하므로 완전자동화는 아님. 자동 마감 처리 옵션이 필요한지 판단 필요.

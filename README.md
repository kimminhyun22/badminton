# 콕매치

배드민턴 동호회·클럽 운영을 위한 모바일 중심 PWA입니다.

- 배포 주소: https://kimminhyun22.github.io/badminton/
- 민턴LIVE: `index.html`
- 팀전LIVE: `team.html`
- 실시간 현황: `view.html`
- 회원용 민턴LIVE 링크: `checkin.html`
- 회원용 팀전LIVE 링크: `rsvp.html`
- 현재 버전: `1.10.396`
- Firebase 프로젝트: `kokmatch-23b31`

## 운영 철학

콕매치는 관리자가 모든 것을 일일이 입력하는 프로그램이 아니라, 회원이 직접 상태를 누르면 운영이 자연스럽게 이어지는 도구입니다.

- 관리자는 상황판을 보고 예외만 처리합니다.
- 회원은 자기 상태를 직접 누릅니다.
- 이미 진행 중인 운동 흐름은 끊지 않습니다.
- 모바일에서 흘끗 봐도 이름, 코트, 다음 대진이 보여야 합니다.
- 설명보다 버튼과 상태칩으로 이해되도록 만듭니다.

## 민턴LIVE

평소 운동용 흐름입니다.

1. 관리자가 클럽 명부를 불러오고 링크를 공유합니다.
2. 회원은 체육관에서 `시작`, `휴식`, `종료` 중 하나를 누릅니다.
3. 초반 자율게임은 `민턴LIVE 시작`에서 진행 중 코트로 등록합니다.
4. 운영 시작 후에는 진행 중 코트, 다음 대진, 예상 대진이 자동으로 이어집니다.
5. 경기가 끝나면 다음 대진 선수가 입장 처리하고, 해당 코트가 새 경기로 바뀝니다.
6. 마칠 때는 `마무리`를 눌러 새 대진 생성을 멈추고 남은 대진만 진행한 뒤 자율게임으로 전환합니다.

### 민턴LIVE 회원 안내 문구

```text
오늘 운동은 콕매치 링크로 진행합니다.
바로 경기 가능하면 [시작], 쉬면 [휴식], 마치면 [종료]만 눌러주세요.
대진은 자동으로 뜨고, 잠깐 필요하면 이번 대진만 뒤로 미룰 수 있습니다.
```

### 민턴LIVE 관리자 체크

- 코트 수가 실제 사용 코트와 맞는지 확인
- 링크 공유 후 충분히 몸을 풀고, 실제 진행 중인 게임만 등록한 뒤 운영 시작
- 진행 중 코트와 다음·예상 대진만 우선 확인
- 초기 등록 게임은 실제로 빨리 끝나도 다음 대진이 입장 처리할 수 있음
- 회원이 휴식·종료를 누르면 다음 자동 보충에 즉시 반영
- 회원 게임신청은 4명이 모두 시작 상태가 되면 우선 반영

## 팀전LIVE

월례 청팀·홍팀 팀전용 흐름입니다.

1. 관리자가 클럽 명부를 선택합니다.
2. 단톡방에 팀전LIVE 링크를 공유합니다.
3. 30분 정도 난타와 현장 정리를 하며 늦게 오는 사람과 게스트를 최종 확인합니다.
4. 관리자는 참가자를 불러오고 필요한 상태·파트너·단장·부단장만 정리합니다.
5. 팀배정 후 청팀·홍팀 균형을 확인합니다.
6. 대진표를 생성하고 품질점수를 확인합니다.
7. 팀전LIVE를 시작하면 같은 링크에서 대진표와 실시간 현황, 승패 입력이 이어집니다.

### 팀전LIVE 회원 안내 문구

```text
팀전LIVE 링크입니다.
참석자는 [출석], 늦게 오면 [늦음]을 눌러주세요.
뒤풀이 참석 여부는 필요한 경우만 표시하면 됩니다.
대진표와 실시간 현황도 같은 링크에서 확인합니다.
```

### 팀전LIVE 관리자 체크

- 불안한 상태에서 바로 대진을 만들지 말고, 현장에서 최종 인원 확인 후 생성
- 늦음 표시 선수는 대진에는 포함하되 라운드 시작 전 대체 가능성을 확인
- 링크 사용이 어려운 회원은 관리자가 출석·늦음·뒤풀이를 대신 변경
- 파트너 요청은 팀배정 전에 처리
- 팀배정 후 단장·부단장과 팀 균형 확인
- 대진 품질점수는 92점 이상이면 바로 진행 가능한 기준으로 봄
- 경기 결과는 승패만 빠르게 입력하고, 서로 다른 입력은 관리자만 확인

## 처음 사용하는 사람을 안심시키는 방법

회원에게 기능을 모두 설명하지 말고, 당장 누를 버튼만 알려주는 것이 좋습니다.

- 민턴LIVE 회원: `시작`, `휴식`, `종료`
- 팀전LIVE 회원: `출석`, `늦음`
- 잘못 눌러도 다시 누르면 바꿀 수 있다고 안내
- 민턴LIVE는 초반 난타와 자율게임을 끊지 말고, 충분히 사람이 모이면 전환
- 팀전LIVE는 링크 공유 후 현장에서 최종 인원과 게스트를 확인한 뒤 대진 생성
- 관리자도 모든 메뉴를 보려 하지 말고 현재 해야 할 버튼과 진행 중/다음 대진만 확인

## 대진 생성 기준

- 실력 균형
- 출전 횟수 공정성
- 최근 파트너 반복 회피
- 최근 상대 반복 회피
- 대기시간 반영
- 진행중 선수 중복 방지
- 팀전 성비 보정

팀전에서 혼복 0은 혼복 금지가 아니라 남복·여복 우선이라는 뜻입니다. 인원 구성상 필요하면 최소한의 혼복 또는 보정경기가 들어갈 수 있습니다.

## 저장과 배포

- 브라우저 자동저장으로 같은 기기에서는 이어서 볼 수 있습니다.
- 중요한 팀전은 JSON 백업을 함께 남기는 것을 권장합니다.
- GitHub Pages 배포는 `main` 브랜치 push 후 보통 1~3분 안에 반영됩니다.
- PWA 캐시 때문에 모바일에서는 새로고침 또는 앱 재실행 후 버전이 바뀔 수 있습니다.

## 실전 테스트 체크리스트

- 민턴LIVE: 명부 불러오기, 링크 공유, 진행 중 코트 등록, 다음 대진 입장, 이번만 뒤로, 마무리 전환
- 팀전LIVE: 명부 불러오기, 링크 공유, 게스트 추가, 참가자 불러오기, 청/홍 배정, 대진 생성, 승패 입력, 전체 초기화
- 현장에서는 버그를 바로 고치려 하지 말고 증상과 상황만 메모한 뒤 컴퓨터에서 확인합니다.

# 콕매치

배드민턴 동호회·모임용 클럽 운동 운영 PWA입니다.

- 배포 주소: https://kimminhyun22.github.io/badminton/
- 실시간 중계 화면: `view.html`
- 현재 버전: `1.10.133`
- Firebase 프로젝트: `kokmatch-23b31`

## 주요 기능

- 참가자 직접 입력 및 명부 관리
- 오늘운동 중심 운영: 클럽 명부를 가져와 모두 대기에서 시작하고, 출석·레슨·휴식·귀가 상태를 반영하며 대기 경기표를 자동 보충
- 월례 팀전 운영: 청/홍팀 자동 배정, 파트너 묶기, 전체 라운드 대진, 대진 품질점검, 재배정
- 선택한 운영 방식은 화면에 활성 상태로 남고, 팀 모드 수동 변경이나 저장본 복원 시에도 자동으로 맞춰집니다.
- 팀전은 상단 `팀전` 탭에서 바로 들어가며, 참가자 입력·청/홍팀 배정·대진 생성·품질점검만 먼저 보이도록 정리합니다.
- 팀전 화면에서는 인당 게임 수, 혼복 횟수, `P+` 파트너 묶기, 청/홍팀 자동 배정, 품질점검, 실시간 중계를 핵심 기능으로 명확히 보여줍니다.
- 팀전 대진 생성 후에는 전체 라운드 대진표를 가장 먼저 보여주고, 코트 진행은 필요할 때 펼쳐 켜는 보조 기능으로 둡니다.
- 팀전 참석 링크는 보조 준비 기능으로 접어두고, 필요할 때만 펼쳐 클럽 명부 선택과 참석자 가져오기를 진행합니다.
- 클럽 명부는 오늘운동과 팀전이 함께 쓰되, 오늘운동 명단과 월례 팀전 참가자 입력은 운영 목적에 맞게 분리합니다.
- 청팀/홍팀 자동 배정과 수동 조정
- 인당 게임 수 기준 자동 대진 생성
- 남복/여복/혼복 우선 배정
- 팀전 성비 불균형 시 보정경기 자동 배정
- 대진 품질 점검
- 승패 입력과 팀 스코어 집계
- 팀전 참석 링크: 클럽 명부를 선택한 뒤 밴드 공지용 링크 하나로 참석 확인, 대진 확정, 실시간 중계까지 연결
- 카카오톡 공유용 실시간 중계 링크
- 오늘운동: 클럽 명부 불러오기, 게스트 표시, 레슨·휴식·귀가·마지막게임·묶음 요청·예약 대진 기반 대기 경기표 자동 생성
- 오늘운동 결과 카드: 경기 종료 시 승팀 기록, 팀전 점수, MVP, 개인 순위, 품질 요약을 한 화면에서 확인
- 팀전 팀별 명단: 청/홍팀 단장·부단장 지정, 팀별 명단 확인, 전체 라운드 대진 진행
- 자동저장, 이름 붙여 저장, JSON 백업
- 모바일 이미지 저장 및 A4 인쇄

## 운영 메모

- `혼복 0`은 혼복 금지가 아니라 남복/여복 우선이라는 뜻입니다.
- 팀전에서 성비가 서로 반대로 남으면 `보정` 경기가 생길 수 있습니다.
- 팀전에서는 경기 중 새 선수 추가를 막고, 기존 선수 제외 후 재배정만 허용합니다.
- 팀전 운영은 `팀전 참석 링크`에서 발송할 클럽 명부를 선택한 뒤 네이버 밴드나 단톡방에 올려 참석·불참·게스트 신청을 먼저 받을 수 있습니다.
- 참석 링크 응답은 관리자 화면에 자동 집계되며, 게스트 제한/신청자 확인 후 `참석자 대진표로 가져오기` 버튼으로 팀전 참가자 입력에 반영합니다.
- 회원은 참석 버튼이 선택 상태로 바뀌는 것을 확인하고, `참석 집계 보기`에서 본인 신청과 전체 참석/게스트 현황을 다시 확인할 수 있습니다.
- 대진표를 생성만 하고 아직 경기 결과를 입력하지 않은 상태라면, 새 참석자를 가져올 때 확인 후 대진표만 초기화하고 참석자 반영을 이어갈 수 있습니다.
- 월례 팀전은 청/홍팀 자동 배정 후 전체 라운드 대진표를 생성하고, 품질점검과 재배정으로 확인한 뒤 진행합니다.
- 팀전 라운드제는 다시 주요 운영 흐름으로 사용합니다. 파트너 매칭, 일부 인원 조정 후 자동 재배정, 대진품질 측정이 모두 전체 대진표 기준으로 동작합니다.
- 팀전의 `P+` 파트너 묶기는 선수 목록에서 한 명을 누른 뒤 함께 묶을 선수를 선택하는 방식이며, 묶인 두 명은 같은 편으로 우선 배정됩니다.
- 팀전에서 `P+` 파트너가 있으면 좋은 후보를 더 많이 비교해, 파트너 고정 때문에 다른 선수의 게임 수가 크게 벌어지는 문제를 줄입니다.
- 콕매치는 오늘운동을 기본 흐름으로 둡니다. 클럽 명부와 상태 입력을 중심으로 코트를 계속 진행합니다.
- 오늘운동은 평소 운동용이고, 월례 팀전은 전체 대진표용으로 분리합니다. 오늘운동의 간단 팀 나눔은 데일리 보조 기능으로만 봅니다.
- 클럽 명부가 여러 개 있으면 선택 전에는 공지 생성을 막아 다른 클럽 회원이 섞이는 실수를 줄입니다.
- 대진 생성 후에는 같은 참석 링크에 `대진표가 준비됨` 상태가 표시되고, 실시간 중계를 켜면 같은 링크에 `실시간 현황 보기` 버튼이 자동으로 붙습니다.
- 참석 링크로 출석을 받은 팀전 운영은 단톡방에 새 중계 링크를 다시 보내지 않고, 처음 보낸 참석 링크를 그대로 안내하는 운영을 권장합니다.
- 참석 링크는 밴드 투표보다 회원에게 약간의 클릭을 요구하므로, 본인 이름 검색을 빠르게 하고 내 중심 정보 확인으로 이어지도록 설계합니다.
- `오늘운동`은 코트가 빌 때마다 다음 4명을 추천하고 임박/예비 대기표를 자동 보충하는 기본 운영 기능입니다.
- 오늘운동 추천 기준은 참여 가능 상태, 팀 실력차, 오늘 경기 수, 대기시간, 반복 파트너/상대, 묶음 요청입니다.
- 오늘운동은 같은 4명이 다시 만나는 조합을 우선 회피하며, 후보가 부족할 때만 경기 생성 안정성을 위해 반복 조합을 허용합니다.
- 오늘운동의 `예약 대진`은 특정 2명을 같은 편으로 넣고 상대를 자동 배정하거나, 특정 4명의 맞대결을 1회성으로 예약하는 기능입니다.
- 예약 대진은 오늘운동의 보조 기능이므로 기본 접힘 상태로 두고, 필요할 때만 펼쳐 사용합니다.
- 예약 대진은 이미 확정된 임박대기를 흔들지 않고, 새 대기표를 채울 때 우선 반영합니다.
- 오늘운동에서 `대기중`은 경기 준비 완료로 해석하며, 코트 수만큼 임박대기와 예비대기 1경기를 자동으로 생성합니다.
- 오늘운동 대기 경기표는 임박대기를 안정적으로 고정하고, 예비대기 1경기만 선수 수정 가능한 완충 구간으로 운영합니다.
- 오늘운동 관리자 화면은 운영 상황판, 진행중 코트, 임박대기, 예비 대기표, 자유 대기자를 먼저 보여주고 전체 선수 상태는 접어둡니다.
- 오늘운동 운영 상황판은 라이브인원과 게스트 수를 함께 보여 현장 인원 변화를 빠르게 확인합니다.
- 오늘운동 운영 상황판은 전체 대기풀과 자유대기자를 분리해, 임박/예비에 들어간 선수까지 포함한 실제 후보 규모를 보여줍니다.
- 오늘운동 `출석 링크 공유`는 카톡방에 회원용 링크를 보내고, 회원이 직접 `대기중/레슨중/휴식중/곧 귀가/마지막게임/종료` 상태를 선택하게 하는 기능입니다.
- 출석 링크에서 들어온 안전한 상태 변경은 자동 반영되고, 경기중이거나 임박/예비 대기표에 포함된 선수의 변경은 관리자 확인 후 반영합니다.
- 회원 출석 링크에는 오늘운동 현황, 진행중 경기, 다음 준비 경기, 간단 팀 점수와 MVP 요약도 함께 표시됩니다.
- 오늘운동 상황판은 출석링크 ON/OFF와 확인 필요한 상태 요청 수를 함께 보여줍니다.
- 회원 출석 화면은 최근 전송 상태를 표시하고, 같은 상태를 짧은 시간에 중복 전송하지 않도록 막습니다.
- 오늘운동에서 게스트는 선수 추가 시 `G` 버튼을 켜고 추가하며, 대기표와 선수 목록에 `G` 배지로 표시됩니다.
- 오늘운동의 `간단 팀 나눔`은 평소 운동 중 임시로 청/홍팀을 나눌 때만 사용하는 보조 기능입니다.
- 오늘운동 결과 카드는 일반 운영에서는 개인 승률/MVP를, 간단 팀 나눔에서는 청/홍팀 점수와 MVP를 함께 보여줍니다.
- 간단 팀 명단 카드에서 청/홍팀 단장과 부단장을 지정할 수 있고, 팀별 인원·성비·실력합을 확인합니다.
- 오늘운동 진행중 코트는 격자 카드로 표시하며, 관리자가 각 코트에서 바로 경기 종료/취소를 처리할 수 있습니다.
- 오늘운동 진행중 코트에서는 `청팀 승 종료`/`홍팀 승 종료`처럼 승팀을 함께 기록할 수 있고, 완료 경기에서도 승패를 보정할 수 있습니다.
- 간단 팀 잠금 상태에서는 가능한 대기표를 미리 보여주되 코트 번호는 배정하지 않습니다. 진행중 경기 종료 버튼을 누르면 안전한 다음 대기표가 방금 빈 코트에 바로 배정됩니다.
- 오늘운동에서 경기 종료를 누르면 방금 끝난 선수들은 대기자로 전환되고 다음 대기표를 자동 보충합니다. 자동배정이 켜져 있을 때만 1순위 임박대기가 해당 코트에 즉시 승계됩니다.
- 자동배정이 꺼져 있으면 완료 후 빈 코트 카드에서 관리자가 1순위 임박대기를 직접 시작합니다. 실전 기본값은 자동배정 OFF를 권장합니다.
- 오늘운동은 코트 수만큼 한 웨이브가 돌면 곧 시작할 1순위와 준비 알림을 보낸 대기표를 보존하고, 나머지 미래 대기표를 오래 기다린 사람 중심으로 다시 섞어 다양성을 확보합니다.
- 오늘운동에서 라이브/대기 인원이 부족하면 가능한 경기 수만 유지하고 상황판에 부족 상태를 표시합니다.
- 오늘운동 완료 경기는 진행중 코트 아래에서 접어두고, 필요할 때만 펼쳐 확인합니다.
- 오늘운동은 임박대기와 자유 대기자를 최우선으로 보여주며, 임박대기자 준비 알림 문구를 공유/복사할 수 있습니다.
- 오늘운동 임박대기에는 다음 코트의 바로 시작/종료임박/예상 종료 상태를 함께 표시합니다.
- 오늘운동의 `마지막게임` 선수는 추천 후보에 포함되며, 경기 완료 후 자동으로 종료 처리됩니다.
- 오늘운동 진행중 경기는 25점 기준 약 15분으로 예상 종료 시간을 표시하고, 남은 5분부터 빨간색 종료임박 상태로 강조합니다.
- 레슨자는 실제 레슨 10분뿐 아니라 준비/정비 시간을 포함해 약 30분 동안 `레슨중` 또는 `휴식중`으로 빼두는 운영을 권장합니다.
- 후반 정리 시간에는 `마감 모드`를 켜면 새 경기 자동 시작과 임박/예비대기 생성을 멈추고 진행중 경기만 마무리할 수 있습니다.
- `마감 예약`에 시각을 입력하면 해당 시각부터 자동으로 마감 모드가 켜집니다. 마감 5분 전부터는 완료 후 자동 승계를 멈춰 실수로 새 경기가 시작되는 것을 줄입니다.
- 오늘운동은 새 날짜에 이전 기록을 발견하면 이어서 볼지, 명단만 남기고 새로 시작할지 묻습니다. 어제 대기시간과 마감모드가 오늘 운영에 섞이지 않게 하기 위함입니다.
- 오늘운동 자동배정은 기본 꺼짐입니다. 실전에서는 추천 결과를 먼저 확인한 뒤 켜는 것을 권장합니다.
- GitHub Pages 배포는 `main` 브랜치 push 후 보통 1~3분 안에 반영됩니다.
- 전체 기능 검증과 실전 케이스별 사용법은 [KOKMATCH_FEATURE_REVIEW.md](KOKMATCH_FEATURE_REVIEW.md)를 참고하세요.
- 오늘운동 전용 외부 리뷰 요청서는 [TODAY_MODE_FABLE_REVIEW.md](TODAY_MODE_FABLE_REVIEW.md)를 참고하세요.

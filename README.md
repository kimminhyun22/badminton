# 콕매치

배드민턴 동호회·클럽 운영을 위한 모바일 중심 PWA입니다.

- 배포 주소: https://kimminhyun22.github.io/badminton/
- 팀전LIVE: `team.html` (기본 시작 화면)
- 민턴LIVE: `index.html`
- 실시간 현황: `view.html`
- 회원용 민턴LIVE 링크: `checkin.html`
- 회원용 팀전LIVE 링크: `rsvp.html`
- 현재 버전: `1.10.437`
- Firebase 프로젝트: `kokmatch-23b31`

## 운영 철학

콕매치는 시스템 관리자가 당일 명단과 코트를 세팅하고, 실제 운동 진행은 클럽 임원이 맡는 반자동 운영 도구입니다.

- 클럽 임원은 코트 종료, 다음 대진 입장·순서 조정, 선수 상태와 파트너 요청을 처리합니다.
- 시스템 관리자는 초기 세팅과 예외 상황만 확인합니다.
- 회원은 자기 경기 확인과 휴식·복귀·종료·뒷풀이 신청만 직접 처리합니다.
- 이미 진행 중인 운동 흐름은 끊지 않습니다.
- 모바일에서 흘끗 봐도 이름, 코트, 다음 대진이 보여야 합니다.
- 430px 이하에서는 진행 코트와 팀전 대진을 한 열로 보여 선수명과 핵심 조작을 우선합니다.
- 설명보다 버튼과 상태칩으로 이해되도록 만듭니다.

## 민턴LIVE

평소 운동용 흐름입니다.

1. 시스템 관리자가 클럽 명부와 코트를 세팅하고 링크를 공유합니다.
2. 클럽 임원이 도착한 회원을 참가 등록하고, 회원은 필요할 때 `휴식`, `복귀`, `종료`를 누릅니다.
3. 초반 자율게임은 `민턴LIVE 시작`에서 진행 중 코트로 등록합니다.
4. 운영 시작 후에는 진행 중 코트, 다음 대진, 예상 대진이 자동으로 이어집니다.
5. 경기가 끝나면 클럽 임원이 `경기 종료`를 누르고 다음 대진을 자동 투입합니다. 부재 선수가 있으면 새 경기 카드에 2분간 보이는 `이번만 뒤로`로 대진을 그대로 미룹니다.
6. 마칠 때는 `마무리`를 눌러 새 대진 생성을 멈추고 남은 대진만 진행한 뒤 자율게임으로 전환합니다.

늦게 도착한 회원은 남은 운동시간이 짧으므로 첫 2경기까지 우선도를 높입니다. 단, 최소 회복시간과 실력 균형을 깨면서 연속 출전시키지는 않습니다. 중도 귀가자는 경기 종료 후 임원에게 알리고 `종료` 처리하면 예비 대진에서 자동 교체되며, 대체 선수가 없을 때만 해당 예비 대진을 다시 만듭니다.

### 민턴LIVE 회원 안내 문구

```text
오늘 운동은 콕매치 링크로 진행합니다.
임원이 참가 등록을 마치면 내 경기를 확인하고, 쉬면 [휴식], 다시 뛰면 [복귀], 마치면 [종료]만 눌러주세요.
뒷풀이 참석은 내 카드에서 직접 신청하고, 파트너 요청은 현장에서 임원에게 말씀해 주세요.
대진은 자동으로 뜹니다. 잠시 자리를 비우면 클럽 임원에게 알려주세요.
```

### 민턴LIVE 클럽 임원 체크

- 코트 수가 실제 사용 코트와 맞는지 확인
- 링크 공유 후 충분히 몸을 풀고, 실제 진행 중인 게임만 등록한 뒤 운영 시작
- 진행 중 코트와 다음·예상 대진만 우선 확인
- 초기 등록 게임이 끝나면 클럽 임원이 `경기 종료`를 눌러 다음 대진을 자동 투입
- 생일축하·공지 때는 `진행 일시 정지`를 눌러 경기 타이머와 다음 순서를 그대로 고정하고, 행사 뒤 `진행 재개`
- 운동 중 늦게 도착한 회원은 임원이 `지각 선수 참가 등록`에서 현장 확인 후 추가
- 선수가 잠시 자리를 비우면 해당 다음 대진 카드의 `이번만 뒤로`를 눌러 4명과 팀 구성을 유지한 채 현재 코트 투입 묶음 뒤로 이동
- 회원이 휴식·종료를 누르면 다음 자동 보충에 즉시 반영
- 파트너 요청은 임원이 두 선수를 선택해 접수하고 필요하면 취소
- 뒷풀이 신청 인원과 이름은 강조 영역에서 확인하고 `명단 복사`로 밴드·단톡방 공지에 붙여넣기
- 시스템 관리자는 운영 요청 충돌이나 연결 복구 같은 예외만 확인

## 팀전LIVE

월례 청팀·홍팀 팀전을 기본으로 하며, 인원이 적거나 팀 구성이 맞지 않을 때는 청·홍 없이 자유 대진으로 운영할 수 있습니다.

1. 관리자가 사전 고지된 오늘 참가자 명단을 확정합니다.
2. 단톡방에 팀전LIVE 링크를 공유합니다.
3. 회원은 링크에서 본인 이름을 확인하면 진행 중인 실중계로 바로 들어갑니다. 별도의 출석 응답은 받지 않습니다.
4. 늦게 오는 선수만 `늦음`을 표시하고, 관리자는 대체 필요 여부를 확인합니다.
5. `청·홍 팀전` 또는 `자유 대진`을 선택합니다. 자유 대진은 청·홍 배정을 건너뛰고 매 경기 실력 균형과 출전 횟수를 맞춥니다.
6. 청·홍 팀전이라면 팀 균형과 파트너·단장·부단장을 확인한 뒤 대진표를 생성하고 품질점수를 확인합니다.
7. 팀전LIVE를 시작하면 같은 링크에서 이름 확인 직후 대진표와 실시간 현황, 승패 입력으로 이어집니다.

### 팀전LIVE 회원 안내 문구

```text
팀전LIVE 링크입니다.
내 이름을 확인하면 실중계로 바로 들어갑니다.
늦게 오는 선수만 [늦음]을 표시해 주세요.
뒤풀이 참석 여부는 필요한 경우만 표시하면 됩니다.
대진표와 실시간 현황도 같은 링크에서 확인합니다.
```

### 팀전LIVE 관리자 체크

- 참가자는 응답으로 모으지 않고 관리자가 사전 확정 명단을 직접 세팅
- 늦음 표시 선수는 대진에는 포함하되 라운드 시작 전 대체 가능성을 확인
- 링크 사용이 어려운 회원은 관리자가 늦음·뒤풀이를 대신 변경
- 파트너 요청은 팀배정 전에 처리
- 팀배정 후 단장·부단장과 팀 균형 확인
- 대진 품질점수는 92점 이상이면 바로 진행 가능한 기준으로 봄
- 경기 결과는 승패만 빠르게 입력하고, 서로 다른 입력은 관리자만 확인

## 처음 사용하는 사람을 안심시키는 방법

회원에게 기능을 모두 설명하지 말고, 당장 누를 버튼만 알려주는 것이 좋습니다.

- 민턴LIVE 회원: `시작`, `휴식`, `종료`
- 팀전LIVE 회원: `내 이름 확인` 후 실중계 자동 진입, 필요할 때만 `늦음`
- 잘못 눌러도 다시 누르면 바꿀 수 있다고 안내
- 민턴LIVE는 초반 난타와 자율게임을 끊지 말고, 충분히 사람이 모이면 전환
- 팀전LIVE는 관리자가 참가자를 먼저 확정하고 늦음·게스트만 확인한 뒤 대진 생성
- 관리자도 모든 메뉴를 보려 하지 말고 현재 해야 할 버튼과 진행 중/다음 대진만 확인

## 대진 생성 기준

- 실력 균형
- 출전 횟수 공정성
- 최근 파트너 반복 회피
- 최근 상대 반복 회피
- 대기시간 반영
- 지각자의 첫 2경기 우선 보정
- 진행중 선수 중복 방지
- 민턴LIVE 개인별 4경기당 혼복 1~2회 목표
- 팀전 성비 보정

팀전에서 혼복 0은 혼복 금지가 아니라 남복·여복 우선이라는 뜻입니다. 인원 구성상 필요하면 최소한의 혼복 또는 보정경기가 들어갈 수 있습니다.

## 저장과 배포

- 브라우저 자동저장으로 같은 기기에서는 이어서 볼 수 있습니다.
- 가대진을 여러 안으로 비교할 때는 상황판 또는 `대진표 생성` 버튼 아래의 `가대진 저장`을 사용합니다. 날짜와 시간이 포함된 이름이 자동 제안되며 최대 10개까지 별도 보관할 수 있습니다.
- `저장 목록`에서 선수 수·게임 수·저장 시각을 비교해 원하는 가대진을 불러올 수 있습니다.
- 중요한 팀전은 JSON 백업을 함께 남기는 것을 권장합니다.
- GitHub Pages 배포는 `main` 브랜치 push 후 보통 1~3분 안에 반영됩니다.
- PWA 캐시 때문에 모바일에서는 새로고침 또는 앱 재실행 후 버전이 바뀔 수 있습니다.

## 실전 테스트 체크리스트

- 민턴LIVE: 명부 불러오기, 링크 공유, 진행 중 코트 등록, 다음 대진 입장, 이번만 뒤로, 마무리 전환
- 팀전LIVE: 관리자 참가자 세팅, 본인 확인 링크, 늦음·대체 확인, 청·홍 팀전/자유 대진 선택, 대진 생성, 승패 입력, 전체 초기화
- 현장에서는 버그를 바로 고치려 하지 말고 증상과 상황만 메모한 뒤 컴퓨터에서 확인합니다.

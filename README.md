# 콕매치

배드민턴 동호회·모임용 클럽 운동 운영 PWA입니다.

- 배포 주소: https://kimminhyun22.github.io/badminton/
- 실시간 중계 화면: `view.html`
- 현재 버전: `1.10.145`
- Firebase 프로젝트: `kokmatch-23b31`

## 주요 기능

- 클럽 명부와 참가자 직접 입력
- 민턴LIVE 중심 운영: 클럽 명부를 가져와 미응답으로 시작하고, 회원 투표와 현장 시작·레슨·휴식·마지막 상태를 반영하며 대기 경기표를 자동 보충
- 출석 링크: 회원이 본인 스마트폰에서 직접 참석·시작·레슨·휴식·마지막 상태를 입력
- 참석투표 마감: 마감 시간을 안내하고, 관리자가 미응답 회원을 제외해 참석자 중심 운영 풀로 정리
- 대기표 자동 보충: 코트 수만큼 임박대기, 예비대기 1경기 유지
- 코트 진행: 경기 종료 시 선수 상태를 대기로 되돌리고 다음 대기표를 자동 보충
- 묶음 요청과 예약 대진: 특정 2명을 같은 편으로 넣거나 특정 4명의 맞대결을 1회성으로 예약
- 회원 게임 신청: 출석 링크에서 파트너 또는 4명 경기 신청을 남기면 관리자가 예약 대진으로 승인
- 승패 입력, 개인 승률, MVP, 품질 요약
- 카카오톡 공유용 민턴LIVE 현황 링크
- 민턴LIVE: 클럽 명부 불러오기, 게스트 표시, 레슨·휴식·마지막·묶음 요청·예약 대진 기반 대기 경기표 자동 생성
- 민턴LIVE 결과 카드: 경기 종료 시 승팀 기록, MVP, 개인 순위, 품질 요약을 한 화면에서 확인
- 자동저장, 이름 붙여 저장, JSON 백업
- 모바일 이미지 저장 및 A4 인쇄

## 운영 메모

- 콕매치는 민턴LIVE를 기본 흐름으로 둡니다. 클럽 명부와 상태 입력을 중심으로 코트를 계속 진행합니다.
- 클럽 명부가 여러 개 있으면 선택 전에는 공지 생성을 막아 다른 클럽 회원이 섞이는 실수를 줄입니다.
- 출석 링크는 밴드 투표보다 회원에게 약간의 클릭을 요구하므로, 본인 이름 검색을 빠르게 하고 내 중심 정보 확인으로 이어지도록 설계합니다.
- `민턴LIVE`는 코트가 빌 때마다 다음 4명을 추천하고 임박/예비 대기표를 자동 보충하는 기본 운영 기능입니다.
- 민턴LIVE 추천 기준은 참여 가능 상태, 팀 실력차, 오늘 경기 수, 대기시간, 반복 파트너/상대, 묶음 요청입니다.
- 민턴LIVE는 같은 4명이 다시 만나는 조합을 우선 회피하며, 후보가 부족할 때만 경기 생성 안정성을 위해 반복 조합을 허용합니다.
- 민턴LIVE의 `예약 대진`은 특정 2명을 같은 편으로 넣고 상대를 자동 배정하거나, 특정 4명의 맞대결을 1회성으로 예약하는 기능입니다.
- 예약 대진은 민턴LIVE의 보조 기능이므로 기본 접힘 상태로 두고, 필요할 때만 펼쳐 사용합니다.
- 예약 대진은 신청 선수들이 모두 대기 가능한 상태가 되면 일반 대진보다 우선 자동 반영됩니다. 이미 확정된 임박대기는 흔들지 않고, 새 대기표를 채울 때 우선 반영합니다.
- 관리자 상황판은 예약대진 자동반영 가능 여부, 빈 코트, 확인 필요한 회원 요청, 대기 부족, 참석투표 마감 상태를 한눈에 보여줍니다.
- 비상 상황에서는 관리자 상황판의 대기표 재생성/예약 우선반영/마감 모드 버튼을 사용하고, 각 대기표 카드에서 순서 변경과 긴급 선수교체를 처리할 수 있습니다.
- 민턴LIVE에서 `시작`은 경기 준비 완료로 해석하며, 코트 수만큼 임박대기와 예비대기 1경기를 자동으로 생성합니다.
- 민턴LIVE 대기 경기표는 임박대기를 안정적으로 고정하고, 예비대기 1경기만 선수 수정 가능한 완충 구간으로 운영합니다.
- 민턴LIVE 관리자 화면은 운영 상황판, 진행중 코트, 임박대기, 예비 대기표, 자유 대기자를 먼저 보여주고 전체 선수 상태는 접어둡니다.
- 민턴LIVE 운영 상황판은 라이브인원과 게스트 수를 함께 보여 현장 인원 변화를 빠르게 확인합니다.
- 민턴LIVE 운영 상황판은 전체 대기풀과 자유대기자를 분리해, 임박/예비에 들어간 선수까지 포함한 실제 후보 규모를 보여줍니다.
- 민턴LIVE `출석 링크 공유`는 카톡방에 회원용 링크를 보내고, 회원이 직접 `참석/시작/레슨/휴식/마지막` 상태를 선택하게 하는 기능입니다.
- 민턴LIVE에서 `미응답`은 링크를 받았지만 아직 누르지 않은 상태입니다. `참석`은 명단에는 남기되 자동대진 후보에서 제외하고, 현장에 도착해 `시작`으로 바뀐 사람만 대진 후보가 됩니다.
- 민턴LIVE에서 `참석투표 마감`을 설정하면 링크 공유 문구와 회원 화면에 마감 시간이 표시됩니다. 관리자는 `참석자만 남기기`로 미응답/불참 회원을 제외할 수 있습니다.
- 회원은 출석 링크에서 파트너 또는 상대까지 지정한 게임 신청을 보낼 수 있습니다. 신청은 바로 대진에 들어가지 않고 관리자 확인 후 예약 대진으로 추가됩니다.
- 출석 링크에서 들어온 안전한 상태 변경은 자동 반영되고, 경기중이거나 임박/예비 대기표에 포함된 선수의 변경은 관리자 확인 후 반영합니다.
- 회원 출석 링크에는 민턴LIVE 현황, 진행중 경기, 다음 준비 경기, MVP 요약도 함께 표시됩니다.
- 민턴LIVE 상황판은 출석링크 ON/OFF와 확인 필요한 상태 요청 수를 함께 보여줍니다.
- 회원 출석 화면은 최근 전송 상태를 표시하고, 같은 상태를 짧은 시간에 중복 전송하지 않도록 막습니다.
- 민턴LIVE에서 게스트는 선수 추가 시 `G` 버튼을 켜고 추가하며, 대기표와 선수 목록에 `G` 배지로 표시됩니다.
- 민턴LIVE 결과 카드는 개인 승률/MVP와 품질 요약을 보여줍니다.
- 민턴LIVE 진행중 코트는 격자 카드로 표시하며, 관리자가 각 코트에서 바로 경기 종료/취소를 처리할 수 있습니다.
- 민턴LIVE 진행중 코트에서는 승팀을 함께 기록할 수 있고, 완료 경기에서도 승패를 보정할 수 있습니다.
- 민턴LIVE에서 경기 종료를 누르면 방금 끝난 선수들은 대기자로 전환되고 다음 대기표를 자동 보충합니다. 자동배정이 켜져 있을 때만 1순위 임박대기가 해당 코트에 즉시 승계됩니다.
- 자동배정이 꺼져 있으면 완료 후 빈 코트 카드에서 관리자가 1순위 임박대기를 직접 시작합니다. 실전 기본값은 자동배정 OFF를 권장합니다.
- 민턴LIVE는 코트 수만큼 한 웨이브가 돌면 곧 시작할 1순위와 준비 알림을 보낸 대기표를 보존하고, 나머지 미래 대기표를 오래 기다린 사람 중심으로 다시 섞어 다양성을 확보합니다.
- 민턴LIVE에서 라이브/대기 인원이 부족하면 가능한 경기 수만 유지하고 상황판에 부족 상태를 표시합니다.
- 민턴LIVE 완료 경기는 진행중 코트 아래에서 접어두고, 필요할 때만 펼쳐 확인합니다.
- 민턴LIVE는 임박대기와 자유 대기자를 최우선으로 보여주며, 임박대기자 준비 알림 문구를 공유/복사할 수 있습니다.
- 민턴LIVE 임박대기에는 다음 코트의 바로 시작/종료임박/예상 종료 상태를 함께 표시합니다.
- 민턴LIVE의 `마지막게임` 선수는 추천 후보에 포함되며, 경기 완료 후 자동으로 종료 처리됩니다.
- 민턴LIVE 진행중 경기는 25점 기준 약 15분으로 예상 종료 시간을 표시하고, 남은 5분부터 빨간색 종료임박 상태로 강조합니다.
- 레슨자는 실제 레슨 10분뿐 아니라 준비/정비 시간을 포함해 약 30분 동안 `레슨중` 또는 `휴식중`으로 빼두는 운영을 권장합니다.
- 후반 정리 시간에는 `마감 모드`를 켜면 새 경기 자동 시작과 임박/예비대기 생성을 멈추고 진행중 경기만 마무리할 수 있습니다.
- `마감 예약`에 시각을 입력하면 해당 시각부터 자동으로 마감 모드가 켜집니다. 마감 5분 전부터는 완료 후 자동 승계를 멈춰 실수로 새 경기가 시작되는 것을 줄입니다.
- 민턴LIVE는 새 날짜에 이전 기록을 발견하면 이어서 볼지, 명단만 남기고 새로 시작할지 묻습니다. 어제 대기시간과 마감모드가 오늘 운영에 섞이지 않게 하기 위함입니다.
- 민턴LIVE 자동배정은 기본 꺼짐입니다. 실전에서는 추천 결과를 먼저 확인한 뒤 켜는 것을 권장합니다.
- GitHub Pages 배포는 `main` 브랜치 push 후 보통 1~3분 안에 반영됩니다.

# 콕매치 기능 검증 및 실전 케이스 정리

검증일: 2026-06-11  
앱 버전: 1.10.133
배포: https://kimminhyun22.github.io/badminton/

이 문서는 콕매치의 전체 기능을 운영자 관점에서 정리하고, 평소 오늘운동과 월례 팀전 운영에서 어떤 케이스를 어떻게 처리하는지 설명합니다.

## 1. 전체 기능 구조

### 운영 모델

- 콕매치는 `오늘운동`과 `월례 팀전`을 성격에 맞게 분리합니다.
- 오늘운동은 `클럽 명부 → 출석/상태 입력 → 대기표 보충 → 코트 진행 → 경기 종료 → 다음 경기 배정 → 실시간 공유` 흐름입니다.
- 월례 팀전은 `참석 링크 → 참가자 확정 → 청/홍팀 배정 → 파트너 묶기/수동 조정 → 전체 라운드 대진 생성 → 품질점검 → 재배정 → 실시간 중계` 흐름입니다.
- 월례 개인전은 오늘운동과 운영 목적이 겹치므로 별도 사용자 프리셋에서 제외하고, 오늘운동 흐름으로 흡수합니다.
- 첫 화면은 오늘운동으로 시작하지만, 월례 팀전은 전체 대진표 화면을 사용합니다.
- 선택한 운영 방식은 활성 카드로 남고, 팀 모드 수동 전환이나 저장본 복원 시에도 자동으로 동기화됩니다.
- 상단 메뉴는 `오늘운동`, `팀전`, `명부`, `설명서` 중심으로 단순화합니다.
- `팀전` 탭은 월례 팀전 전용 진입점입니다. 참가자 입력, 청/홍팀 배정, 전체 대진 생성, 품질점검을 먼저 보여주고 참석 링크 같은 준비 도구는 접어둡니다.
- 팀전 화면은 기존 핵심 기능인 인당 게임 수 보장, 혼복 횟수 설정, `P+` 파트너 묶기, 청/홍팀 자동 배정, 품질점검, 실시간 중계를 명확히 노출합니다.
- 팀전 대진 생성 후에는 품질점검을 먼저 보여주고, 이어서 승패 집계와 코트 진행을 확인한 뒤 전체 라운드 대진표를 봅니다.
- 오늘운동의 간단 팀 나눔은 평소 운동 중 임시로 팀을 나눌 때만 쓰는 보조 기능입니다.

### 월례 팀전 운영

- 대진표 참가자 입력을 기준으로 청/홍팀을 자동 배정합니다.
- 발송할 클럽 명부를 선택한 뒤 네이버 밴드/단톡방에 `팀전 참석 링크`를 공유해 참석·불참·게스트 신청을 먼저 받을 수 있습니다.
- 참석 링크 응답은 관리자 화면에 자동 집계되고, 게스트 제한/신청자 확인 후 참석 확정자와 게스트를 대진표 참가자로 가져올 수 있습니다.
- 참석자는 버튼 선택 상태와 참석 집계 보기로 본인 신청이 저장됐는지 다시 확인할 수 있습니다.
- 생성만 하고 아직 진행하지 않은 대진표는 새 참석자 반영 시 확인 후 자동 초기화하고 업무를 이어갈 수 있습니다.
- 대진표가 생성되면 같은 참석 링크에 `대진표 준비됨` 상태가 표시되고, 실시간 중계가 시작되면 같은 링크에서 실시간 현황으로 이동할 수 있습니다.
- 전체 라운드 대진표를 생성한 뒤 품질점검에서 실력 균형, 출전 공정성, 파트너 반복, 재대결, 휴식 간격, 설정 준수를 확인합니다.
- 결과 요약에서 청/홍팀 점수, 승패 미입력 경기, MVP, 개인 순위, 품질 요약을 함께 확인합니다.
- 회원 출석 링크도 오늘운동 현황판 역할을 겸해 진행중 경기, 다음 준비 경기, 팀전 점수와 MVP를 간단히 보여줍니다.
- 팀전 명단 카드에서 청/홍팀 단장·부단장을 지정하고, 팀별 인원·성비·실력합을 확인합니다.
- 라운드제 전체 대진을 기준으로 운영합니다. 안정성이 확인되면 빠른 진행 기능으로 빈 코트에 안전한 다음 경기를 넣을 수 있습니다.
- `혼복 0`은 혼복 금지가 아니라 남복/여복 우선입니다. 불가피하면 혼복 또는 보정 경기를 허용해 전체 공정성을 맞춥니다.
- 대진 품질 점검에서 실력 균형, 출전 공정성, 파트너 반복, 재대결, 휴식 간격, 설정 준수 여부를 확인합니다.
- 경기 중 일부 라운드까지 진행한 뒤 재배정할 수 있습니다.
- 대진 생성 후 라운드 단위로 전체 대진을 확인하고, 필요 시 코트별 종료 버튼으로 완료 처리해 현재 진행 중인 다른 코트 선수와 겹치지 않는 다음 경기를 바로 배정할 수 있습니다.
- 팀전에서는 실전 혼란을 막기 위해 경기 중 새 선수 투입을 막고, 기존 선수 제외 후 재배정만 허용합니다.
- 카카오톡 링크로 공유할 수 있는 실시간 중계 화면 `view.html`을 제공합니다.

### 오늘운동 모드

- 목표 게임 수를 먼저 고정하지 않고, 코트가 빌 때마다 다음 대기 경기를 운영하는 기본 기능입니다.
- 선수 상태는 `대기중`, `경기중`, `레슨중`, `휴식중`, `곧 귀가`, `마지막게임`, `종료`로 관리합니다.
- `대기중`은 바로 경기할 준비가 됐다는 뜻입니다.
- `출석 링크 공유`로 회원이 본인 스마트폰에서 직접 상태를 보낼 수 있습니다.
- 경기중 또는 임박/예비 대기표에 들어간 선수의 요청은 자동 반영하지 않고 관리자 확인 대상으로 남깁니다.
- 코트 수만큼 임박대기를 만들고, 예비대기는 1경기만 유지합니다.
- 예비대기는 너무 멀리 확정하지 않아 새로 온 사람, 방금 끝난 사람, 레슨 복귀자를 반영할 여지를 남깁니다.
- 진행중 코트는 격자 카드로 표시하고, 관리자가 코트별로 바로 경기 종료/취소할 수 있습니다.
- 경기 종료 시 승팀을 함께 기록할 수 있고, 완료 후에도 관리자 화면에서 승패를 보정할 수 있습니다.
- 오늘운동 결과 카드에는 개인 승률/MVP와 품질 요약이 표시됩니다.
- 경기 종료를 누르면 방금 끝난 선수는 다시 대기자로 돌아오고 대기표가 자동 보충됩니다.
- 오늘운동의 간단 팀 잠금 상태에서는 경기 종료 버튼을 누르면 현재 경기 중인 선수와 겹치지 않는 다음 대기표가 방금 빈 코트에 바로 들어갑니다.
- 자동배정 ON일 때만 1순위 임박대기가 해당 코트에 즉시 승계됩니다. 자동배정 OFF에서는 빈 코트 카드에서 관리자가 직접 시작합니다.
- 예약 대진으로 특정 2명을 같은 편으로 묶거나, 특정 4명의 맞대결을 1회성으로 예약할 수 있습니다.
- 마감 예약을 설정하면 지정 시각부터 새 경기 생성을 멈추고 진행중 경기만 마무리합니다.

## 2. 실전 운영 케이스

### 케이스 A: 오늘운동 기반 일반 운영

1. `오늘운동`에서 클럽 명부를 가져옵니다.
2. 단톡방에 출석 링크를 올려 회원이 본인 상태를 직접 입력하게 합니다.
3. 관리자는 라이브 인원, 임박대기, 예비대기, 자유 대기자를 확인합니다.
4. 레슨중·휴식중·곧 귀가·마지막게임 상태를 반영합니다.
5. 코트가 끝나면 해당 코트의 종료 버튼을 눌러 다음 임박대기를 시작합니다.
6. 묶음 요청이나 예약 대진이 있으면 예비대기 생성 전에 반영합니다.
7. 마감 시간이 가까워지면 마감 모드 또는 마감 예약으로 새 경기 생성을 멈춥니다.

주의할 점:

- 코트 진행은 진행 중인 다른 코트 선수와 겹치는 경기를 배정하지 않습니다.
- 대기 인원이 부족하면 다양성보다 경기 생성 안정성이 우선됩니다. 이때는 예비대기를 적게 유지하고 관리자가 직접 시작하는 편이 안전합니다.
- 출석 링크를 이미 공유한 뒤 명부가 바뀌면 링크 재공유로 최신 안내를 확인합니다.
- 클럽 명부가 여러 개 있을 때 선택하지 않으면 공지 생성을 막아 다른 클럽 회원이 섞이지 않게 합니다.
- 참석 링크로 출석을 받은 경우, 실시간 중계 시작 후 단톡방에는 새 링크 대신 처음 참석 링크를 다시 안내하는 것이 사용자 혼란이 적습니다.

### 케이스 B: 팀전 운영

1. `팀전 참석 링크`로 참석 확정자와 게스트를 받습니다.
2. `참석자 대진표로 가져오기`로 참가자 입력에 반영합니다.
3. 파트너 묶기, 청/홍팀 자동 배정, 단장·부단장 지정, 팀원 수동 조정을 진행합니다.
4. 전체 라운드 대진표를 생성하고 품질점검을 확인합니다.
5. 품질이 낮으면 일부 인원 조정 또는 재배정을 진행합니다.
6. 현장에서는 라운드제 전체 대진을 기준으로 진행하고, 필요하면 빠른 진행으로 빈 코트에 안전한 다음 경기를 넣습니다.

확인할 핵심 설정:

- `인당 게임 수`: 모든 선수의 최소 출전 목표입니다.
- `혼복 인당 횟수`: `0`이면 남복/여복 우선, 불가피하면 최소 보정만 허용합니다.
- `P+`: 지정한 두 명을 같은 편으로 우선 묶습니다.
- `P+`가 있으면 후보 탐색을 더 많이 해 파트너 고정과 전체 게임 수 공정성을 함께 맞춥니다.
- `실시간 중계`: 대진표와 팀 점수, MVP/개인 순위를 공유 화면으로 보냅니다.

운영 철학:

- 팀전 중간 투입은 막는 것이 맞습니다. 실전에서는 팀 점수, 선수 게임 수, 성비, 승패 집계가 동시에 꼬일 수 있습니다.
- 새 사람이 오면 팀전이 아닌 오늘운동 운영으로 전환하거나, 다음 행사부터 반영하는 쪽이 안정적입니다.

### 케이스 C: 성비가 서로 반대로 남는 팀전

예시:

- 청팀: 남2, 여1
- 홍팀: 남1, 여2

검증 결과:

- 청팀 잔여 목표: 남 8, 여 4
- 홍팀 잔여 목표: 남 4, 여 8
- 보정 필요 판단: 정상
- 보정 경기 후보 생성: 정상
- 보정 경기 타입: `보정`

의미:

- 한 팀은 남자 출전 슬롯이 더 필요하고, 다른 팀은 여자 출전 슬롯이 더 필요합니다.
- 이 경우 남복/여복/정상 혼복만으로는 공정하게 채우기 어려워 `여1 남3` 또는 `여3 남1` 형태의 보정 경기가 필요할 수 있습니다.

### 케이스 D: P 파트너 고정

사용 상황:

- 특정 두 명이 혼복 연습을 원함
- 파트너와 꼭 같은 편으로 한두 게임 하고 싶음

검증 결과:

- 16명, 인당 4게임 조건에서 P 파트너 1쌍을 고정했습니다.
- 파트너 분리 위반: 0건
- 해당 파트너쌍 출전: 4게임

주의할 점:

- P 파트너는 강한 조건입니다.
- 파트너 고정을 많이 걸면 전체 품질 점수가 낮아질 수 있습니다.
- 실전에서는 꼭 필요한 요청만 P로 묶는 것이 좋습니다.

### 케이스 E: 오늘운동 3코트, 30명 안팎, 레슨 포함

가정:

- 운동 시간: 19:00~22:00
- 운영 마감: 21:00
- 3코트 사용
- 참석자는 30명 안팎
- 12~15명이 레슨을 받으며, 레슨 10분 전후 준비/정비 시간까지 약 30분 제외
- 21:00 이후는 자율게임 전환

권장 운영:

1. 19:00 전후 명단을 불러오고 출석자를 `대기중`으로 둡니다.
2. 레슨자는 `레슨중` 또는 `휴식중`으로 빼둡니다.
3. 코트 3개 기준 임박대기 3경기와 예비대기 1경기만 유지합니다.
4. 경기가 끝난 코트에서 `경기 종료`를 누릅니다.
5. 1순위 임박대기가 자동 입장하고, 방금 끝난 선수는 다시 대기자로 돌아옵니다.
6. 21:00 마감 예약을 걸어두면 이후 새 경기 생성을 멈춥니다.

검증 결과:

- 6개 시드 스트레스 테스트
- 평균 완료 경기: 20.8경기
- 평균 팀 실력차: 0.93
- 최대 팀 실력차: 4.7
- 예약 대진 생성: 12건
- 예약 대진 실제 시작: 9건
- 마감 이후 새 경기 시작: 0건
- 대기표 정합성 오류: 0건

해석:

- 예약이 늦게 생성되거나 선수가 레슨/귀가로 빠지면 모든 예약이 성사되지는 않습니다.
- 하지만 가능한 예약은 우선 반영되고, 마감 이후 새 경기가 시작되는 문제는 차단됐습니다.
- 오늘운동은 전체를 완벽히 미리 확정하는 방식보다, 현장 변화에 맞춰 짧게 확정하고 계속 갱신하는 방식이 맞습니다.

### 케이스 F: 묶기 예약과 4명 맞대결 예약

묶기 예약:

- A와 B를 같은 편으로 묶고 상대는 자동 배정합니다.

운영 원칙:

## 4. 남은 주의점과 개선 후보

### 실전 전 주의점

- 오늘운동 자동배정은 기본 꺼짐으로 시작하는 것이 안전합니다.
- 실전 첫 운영에서는 관리자가 임박대기와 예비대기를 눈으로 확인한 뒤 시작 버튼을 누르는 방식을 권장합니다.
- 예약 대진은 선수가 레슨중, 경기중, 귀가 상태이면 바로 반영되지 않을 수 있습니다.
- 마감 예약은 운영 종료 시각을 명확히 정한 클럽에서 특히 유용합니다.

### 개선 후보

- 18명, 여6/남12 같은 일부 성비에서 단일 조건으로는 품질 C가 나올 수 있습니다. 향후 보완 대상입니다.

- 출석/대기/마지막게임을 선수 개인이 직접 입력하는 링크형 기능은 다음 단계 핵심 후보입니다.
- 레슨 스케줄을 미리 등록하고 자동으로 `레슨중` 처리하는 기능도 현실성이 높습니다.

## 5. 추천 운영 방식

### 주말 월례대회

- 개인전 또는 팀전으로 전체 대진표를 먼저 생성합니다.
- 품질 A 이상이면 그대로 운영합니다.
- B 이하면 실전 특이사항을 보고 재생성 또는 설정 조정합니다.
- 대진 생성 후 자동으로 켜지는 코트 진행 카드에서 코트별 `승 종료`를 눌러 끝난 코트부터 바로 다음 경기를 넣습니다.
- 팀전 중간 투입은 하지 않습니다.

### 평일 데일리 운동

- 오늘운동 모드에서 출석자를 빠르게 넣습니다.
- 레슨자는 레슨 전후 준비시간까지 빼둡니다.
- 임박대기 3팀 + 예비대기 1팀 구조로 운영합니다.
- 코트가 끝날 때마다 관리자가 경기 종료를 누릅니다.
- 21:00 같은 운영 마감 시각을 예약하고 이후는 자율게임으로 넘깁니다.

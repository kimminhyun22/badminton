# 콕매치 민턴LIVE 기능 검증 및 실전 케이스 정리

검증일: 2026-06-13
앱 버전: 1.10.142
배포: https://kimminhyun22.github.io/badminton/

이 문서는 현재 콕매치의 사용자-facing 방향을 `민턴LIVE` 중심으로 정리합니다. 팀전 기능은 화면에서 걷어내고, 평소 클럽 운동 운영에 필요한 출석, 상태, 대기표, 코트 진행, 실시간 공유를 먼저 안정화합니다.

## 1. 현재 운영 모델

- 기본 흐름은 `클럽 명부 → 참석투표 → 현장 상태 입력 → 임박대기/예비대기 생성 → 코트 진행 → 경기 종료 → 다음 경기 배정 → 실시간 공유`입니다.
- 회원은 출석 링크에서 본인 상태를 직접 보낼 수 있습니다.
- 관리자는 운영 상황판에서 라이브 인원, 게스트, 진행중 코트, 임박대기, 예비대기, 자유 대기자를 확인합니다.
- `미응답`은 아직 참석 의사를 밝히지 않은 상태입니다.
- `참석예정`은 운동에 올 예정이지만 아직 현장 대기자가 아니므로 자동 대진 후보에서 빠집니다.
- `대기중`은 바로 경기할 준비가 됐다는 뜻입니다.
- `레슨중`, `휴식중`, `종료`, `경기중` 선수는 자동 대진 후보에서 빠집니다.
- `곧 귀가`, `마지막게임` 선수는 후보에 포함하되 우선순위를 조정합니다.
- 코트 수만큼 임박대기를 만들고 예비대기는 1경기만 유지합니다.
- 예비대기를 과하게 확정하지 않아 새로 온 사람, 방금 끝난 사람, 레슨 복귀자를 반영할 여지를 남깁니다.

## 2. 관리자 사용 흐름

1. `민턴LIVE`에서 클럽 명부를 가져옵니다.
2. 참석투표 마감 시간을 설정하고 카톡방에 출석 링크를 공유합니다.
3. 회원이 먼저 `참석예정` 또는 `대기중`으로 참석 의사를 남깁니다.
4. 마감 후 관리자는 `참석자만 남기기`로 미응답 회원을 제외합니다.
5. 회원이 현장 상황에 맞게 `대기중/레슨중/휴식중/곧 귀가/마지막게임/종료`를 선택합니다.
6. 필요한 경우 회원이 파트너 또는 4명 경기 신청을 보내고, 관리자는 예약 대진으로 승인합니다.
7. 관리자는 운영 상황판에서 실제 라이브 인원과 자유 대기자를 봅니다.
8. 임박대기가 만들어지면 선수 이름과 코트 준비 상태를 확인합니다.
9. 코트가 끝나면 진행중 코트 카드에서 `경기 종료`를 누릅니다.
10. 자동배정이 켜져 있으면 1순위 임박대기가 빈 코트로 들어갑니다.
11. 자동배정이 꺼져 있으면 관리자가 빈 코트 카드에서 직접 시작합니다.
12. 마감 시간이 가까워지면 마감 모드 또는 마감 예약으로 새 경기 생성을 멈춥니다.

## 3. 회원 사용 흐름

1. 단톡방 링크를 누릅니다.
2. 본인 이름을 빠르게 찾습니다.
3. 참석 의사를 먼저 선택합니다.
4. 현장에 도착하면 `대기중`, 레슨이면 `레슨중`, 쉬는 중이면 `휴식중`처럼 상태를 바꿉니다.
5. 원하는 파트너나 맞대결이 있으면 게임 신청을 보냅니다.
6. 본인이 대기표에 들어갔는지, 다음 경기인지, 진행 상황이 어떤지 확인합니다.
7. 마지막 게임 또는 귀가 예정이면 상태를 바꿔 관리자가 대체 인원을 볼 수 있게 합니다.

## 4. 주요 기능

### 출석 링크

- 회원이 직접 상태를 입력합니다.
- 참석투표 마감 시간을 표시할 수 있습니다.
- 마감 후 미응답 회원은 더 이상 직접 참석으로 바꾸지 못하고, 관리자가 필요 시 현장에서 처리합니다.
- 관리자는 `참석자만 남기기`로 미응답 회원을 운영 명단에서 제외할 수 있습니다.
- 경기중이거나 임박/예비 대기표에 들어간 선수의 상태 변경은 관리자 확인 대상으로 남깁니다.
- 링크는 민턴LIVE 현황판 역할도 겸합니다.

### 대기표 자동 보충

- 임박대기는 코트 수만큼 유지합니다.
- 예비대기는 1경기만 유지합니다.
- 같은 4명 반복과 같은 파트너/상대 반복을 줄입니다.
- 오늘 적게 친 사람, 오래 기다린 사람, 곧 가는 사람을 우선 고려합니다.
- 후보가 부족하면 다양성보다 경기 생성 안정성을 우선합니다.

### 예약 대진

- 특정 2명을 같은 편으로 묶고 상대를 자동 배정할 수 있습니다.
- 특정 4명의 맞대결을 1회성으로 예약할 수 있습니다.
- 회원이 링크에서 보낸 게임 신청은 관리자 승인 후 예약 대진으로 들어갑니다.
- 예약 대진은 임박대기를 흔들지 않고 새 대기표를 채울 때 우선 반영합니다.
- 예약 선수가 레슨중, 경기중, 귀가 상태이면 바로 반영되지 않을 수 있습니다.

### 코트 진행

- 진행중 코트는 격자 카드로 표시합니다.
- 관리자는 코트별로 경기 종료, 취소, 승패 보정을 처리합니다.
- 경기 종료 시 방금 끝난 선수는 다시 대기자로 돌아갑니다.
- `마지막게임` 선수는 경기 완료 후 자동 종료 처리됩니다.

### 마감 모드

- 마감 모드가 켜지면 새 경기 자동 시작과 대기표 생성을 멈춥니다.
- 마감 예약 시각이 되면 자동으로 마감 모드가 켜집니다.
- 마감 5분 전부터는 실수로 새 경기가 시작되지 않도록 보수적으로 운영합니다.

## 5. 실전 케이스

### 3코트, 30명 안팎, 레슨 포함

가정:

- 운동 시간: 19:00~22:00
- 운영 마감: 21:00
- 3코트 사용
- 참석자는 30명 안팎
- 12~15명이 레슨을 받으며, 레슨 전후 준비/정비 시간을 포함해 약 30분 제외

권장 운영:

1. 19:00 전후 명단을 불러오고 출석자를 `대기중`으로 둡니다.
2. 레슨자는 `레슨중` 또는 `휴식중`으로 빼둡니다.
3. 코트 3개 기준 임박대기 3경기와 예비대기 1경기만 유지합니다.
4. 경기가 끝난 코트에서 `경기 종료`를 누릅니다.
5. 방금 끝난 선수는 다시 대기자로 돌아오고, 대기표를 다시 보충합니다.
6. 21:00 마감 예약을 걸어두면 이후 새 경기 생성을 멈춥니다.

검증 요약:

- 대기표 정합성 오류: 0건
- 마감 이후 새 경기 시작: 0건
- 예약 대진은 가능한 경우 우선 반영
- 후보 부족 시 가능한 경기 수만 유지

## 6. 테스트 체크리스트

- `index.html` 스크립트 문법 확인
- `view.html` 스크립트 문법 확인
- `checkin.html` 출석 링크 동작 확인
- 명부 가져오기 후 중복 선수 방지 확인
- 상태 변경이 대기표 후보에 반영되는지 확인
- 진행중/임박/예비 선수의 잠금 상태 확인
- 경기 종료 후 선수 상태와 대기표 보충 확인
- 마감 모드와 마감 예약 확인
- 모바일 화면에서 이름, 코트번호, 다음 경기 가독성 확인

## 7. 남은 개선 후보

- 출석 링크에서 회원 본인 중심 정보 강화
- 레슨 시간표 사전 등록과 자동 상태 변경
- 게스트 신청 제한과 신청자 확인 UX 정리
- 운영 상황판에서 “지금 당장 확인해야 할 것”을 더 강하게 강조
- 실제 클럽 운영 피드백을 기반으로 자동배정 기본값 조정

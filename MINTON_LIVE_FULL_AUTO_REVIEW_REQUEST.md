## 1. 제품 철학

민턴LIVE는 관리자가 일일이 승인하고 대진을 짜는 시스템이 아니라, 회원 입력을 기반으로 자동 운영되는 오토시스템이어야 해.

핵심 원칙:

- 회원이 링크에서 `참석`, `시작`, `휴식`, `종료`를 직접 누른다.
- 회원이 링크에서 파트너/게임 신청을 직접 한다.
- 관리자는 최소한만 개입한다.
- 게임신청은 관리자 승인 대상이 아니라 자동 접수/자동 반영 대상이다.
- 경기 중인 선수만 즉시 변경하지 않고 예약 또는 경기 종료 후 반영한다.
- 경기 중이 아닌 선수는 임박대기 포함 여부와 관계없이 상태 변경/선수 변경/대진 변경이 즉시 반영되어야 한다.
- 중요한 것은 자동화 속도뿐 아니라 대진밸런스, 공정성, 다양성이다.

## 2. 최근 반영된 핵심 변경

아래 변경이 실제로 잘 작동하는지 검토해줘.

### 게임신청 자동화

- 회원이 `checkin.html`에서 파트너/게임 신청을 보내면 관리자 승인 없이 자동 처리.
- 가능한 신청은 `_dailyRegisterReservationRequest()`에서 자동 등록.
- 대기표 반영 가능하면 `_dailyPromoteReadyReservations(true)`로 자동 반영.
- 불가능한 신청은 관리자 확인 목록에 올리지 않고 자동 무시 처리.
- 관리자 대시보드의 `처리 필요` 카운트에는 게임신청이 포함되지 않아야 함.

### 경기 중/임박대기 처리

- 경기 중인 선수는 파트너/게임 신청은 예약으로 등록되고, 즉시 대진에는 들어가지 않음.
- 경기 종료 후 해당 선수가 `시작` 상태가 되면 예약 대진으로 자동 반영되어야 함.
- 임박대기는 임시 편성으로 보고, 회원이 휴식/종료를 누르거나 게임신청에 포함되면 즉시 대진에서 빠지고 재편성되어야 함.

### 같은 편 신청의 상대 자동선정

회원이 상대팀을 직접 지정하지 않고 같은 편만 신청한 경우:

- 우리팀 남남이면 상대도 남남
- 우리팀 여여이면 상대도 여여
- 우리팀 남녀이면 상대도 남녀
- 같은 형태의 상대 후보가 없으면 억지 예외 대진을 만들지 않고 대기

## 4. 반드시 검토할 질문

아래 순서대로 답변해줘.

1. P0/P1 버그가 있는가?
   - 회원 상태 변경, 게임신청, 자동 대진, 경기 종료 후 자동 보충 중 실전에서 막히는 오류가 있는가?

2. 회원 게임신청은 정말 오토시스템인가?
   - 관리자 승인 없이 자동 접수되는가?
   - 가능한 신청은 자동 대기표로 들어가는가?
   - 불가능한 신청은 관리자에게 보고되지 않고 안전하게 처리되는가?
   - 회원 입장에서는 신청이 접수됐다는 인지가 충분한가?

3. 경기 중 선수 처리 방식이 맞는가?
   - 경기 중인 선수에게 파트너/게임신청이 들어오면 예약으로 남는가?
   - 경기 종료 후 자동 반영되는가?
   - 경기 중 선수가 휴식/종료를 누르면 경기 종료 후 반영되는가?

4. 임박대기 처리 방식이 맞는가?
   - 임박대기는 임시 편성으로 보고 자유롭게 바뀌는가?
   - 임박대기 선수가 휴식/종료를 누르면 즉시 빠지는가?
   - 임박대기 선수가 게임신청에 포함되면 기존 임시 대진에서 빠지고 신청 대진이 우선되는가?

5. 대진밸런스와 공정성은 충분한가?
   - 같은 편 신청이 있을 때도 실력차가 과하게 벌어지지 않는가?
   - 같은 사람/같은 상대/같은 파트너 반복이 과도하지 않은가?
   - 적게 뛴 사람, 오래 기다린 사람이 우선 반영되는가?
   - 소인원/후반부에는 어떤 한계가 있는가?

6. 같은 편 신청의 상대 자동선정이 맞는가?
   - 남복 신청이면 상대도 남복
   - 여복 신청이면 상대도 여복
   - 혼복 신청이면 상대도 혼복
   - 맞는 상대 후보가 없을 때 대기 처리하는 것이 적절한가?

7. 관리자 UX가 완전자동화에 맞는가?
   - 관리자가 확인해야 하는 정보와 자동으로 흘러가는 정보가 구분되는가?
   - 게임신청이 처리 필요처럼 보이지 않는가?
   - 상단 대시보드만 보고 특이사항을 판단할 수 있는가?
   - 불필요한 버튼/문구/접힘 구조가 남아 있는가?

8. 회원페이지 UX가 충분히 단순한가?
   - 본인 이름 찾기, 참석/시작/휴식/종료, 게임신청 흐름이 명확한가?
   - 40~50대가 스마트폰으로 운동 중 볼 때 이름/코트/다음 투입이 잘 보이는가?
   - 파트너/게임신청 폼이 헷갈리지 않는가?

## 5. 시뮬레이션 요청

가능하면 코드 근거로 아래 상황을 시뮬레이션해줘.

1. 30명, 4코트, 정상 운영
2. 30명, 3코트, 중간 휴식/종료/복귀 섞임
3. 18명, 3코트, 대기 인원이 부족한 후반부
4. 경기 중인 선수를 파트너로 신청
5. 임박대기 선수를 파트너로 신청
6. 남복/여복/혼복 같은 편 신청에서 상대 자동선정
7. 4명 경기 신청에서 1명이 휴식/종료를 누르는 경우
8. 동시에 여러 회원이 신청하는 경우
9. 코트가 동시에 여러 개 비는 경우
10. 자동 시작 on/off 각각의 흐름

각 케이스마다 확인할 것:

- 중복 배정 없음
- 진행 중 선수 중복 투입 없음
- 대기표 깨짐 없음
- 대진밸런스 과도한 악화 없음
- 자동 반영 실패 시 관리자에게 불필요한 업무가 생기지 않음
